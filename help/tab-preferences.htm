Sleeptime

The sleeptime setting is used to configure how often Belvedere will rotate through the folder and rules. After each process time, Belevedere will go to sleep for the specified period of time. Once this time period as passed, Belvedere will 'wake-up' and work through all the active rules for each folder. All times inserted are in milliseconds.

Examples:

  • 3000000 - 3 hours
  • 300000 - 3 minutes
  • 30000 - 30 seconds
  • 3000 - 3 seconds

System Logging

Belvedere is capable of logging all the actions within the system in 3 different forms:

  1. System Events (System) - This option allows the user to log all activity within Belvedere with respect to option changes, rule creation/deletion/activation.
  2. System Actions (Actions) - this opiton allows the user to log all actions that Belvedere takes based upon the rules that are setup and active within the system.
  3. All System Occurances (Both) - This option is a combination of both of the options listed above.

Examples:

    System Events

    2009-04-28 10:38:00 Logging has been enabled with type: Both

    2009-05-01 03:28:06 SYSTEM Belvedere has been paused

    System Actions

    2009-04-28 10:38:14 ======================================
    2009-04-28 10:38:14 Action taken: Move file
    2009-04-28 10:38:14 File: C:\Documents and Settings\test\test.pdf
    2009-04-28 10:38:14 Destination: C:\Documents and Settings\test\Desktop\pdf
    2009-04-28 10:38:14 ======================================
